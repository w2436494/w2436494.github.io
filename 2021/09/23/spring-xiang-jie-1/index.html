１、spring

1.1、简介

Spring框架是由于软件开发的复杂性而创建的。Spring使用的是基本的JavaBean来完成以前只可能由EJB完成的事情。然而，Spring的用途不仅仅限于服务器端的开发。从简单性、可测试性和松耦合性角度而言，绝大部分Java应用都可以从Spring中受益。

  • 目的：解决企业应用开发的复杂性
  • 功能：使用基本的JavaBean代替EJB，并提供更多的企业应用功能
  • 范围：任何Java应用

spring是一个轻量级控制反转(IOC)和面向切面的容器框架。

1.2、优点

  • spring是一个开源的免费框架（容器）

  • spring是一个轻量级，非入侵式的框架。

    • 轻量级：引入一个框架需要引入很多jar包，但spring体积比较小（jar比较少），且引入过程中不需要其他的组件可以单独使用

    • 非入侵式的：引入spring不会改变代码原来的任何情况

    • 入侵式的：导入一个jar包，项目不能使用

  • 控制反转（IOC），面向切面编程（AOP）

  • 支持事务的处理

  • 对框架整合的支持，几乎市面上的Java框架都能整合进去

总结一句话：Spring就是一个轻量级的控制反转（IOC）和面向切面编程的框架。

2、IOC理论推导

原来的业务开发

1、UserDao接口

2、UserDaoImpl实现类

3、UserService业务接口

4、UserServiceImpl业务实现类

package com.wang.dao;

public interface UserDao {
    void getUser();
}

package com.wang.dao;

public class UserDaoImpl implements UserDao{

    private UserDao userDao = new UserDaoImpl();
    @Override
    public void getUser() {
        System.out.println("获取默认用户的数据");
    }
}
package com.wang.service;

public interface UserService {
    void getUser();
}


package com.wang.service;

import com.wang.dao.UserDao;
import com.wang.dao.UserDaoImpl;

public class UserServiceImpl implements UserService{
    // 业务层调用dao层
    // 组合
    private UserDao userDao = new UserDaoImpl();
    @Override
    public void getUser() {
        userDao.getUser();
    }
}

// 测试类
import com.wang.service.UserService;
import com.wang.service.UserServiceImpl;

public class MyTest {

    public static void main(String[] args) {
        // 用户实际调用的是业务层，dao层不需要其接触
        UserService userService = new UserServiceImpl();
        userService.getUser();
    }
}

如果现在有一个新的需求，需要使用mysql/oracle实现获取用户数据

需要新建一个类

package com.wang.dao;

public class UserDaoMysqlImpl implements UserDao {
    @Override
    public void getUser() {
        System.out.println("Mysql获取默认用户的数据");
    }
}

修改业务层代码

package com.wang.service;

import com.wang.dao.UserDao;
import com.wang.dao.UserDaoImpl;
import com.wang.dao.UserDaoMysqlImpl;

public class UserServiceImpl implements UserService{
    // 业务层调用dao层
    // 组合
    private UserDao userDao = new UserDaoMysqlImpl();
    @Override
    public void getUser() {
        userDao.getUser();
    }
}

当这种需求增多时会导致大量变更，当用户需求改变就要去该源码。需要用到设计模式

package com.wang.service;

import com.wang.dao.UserDao;

public class UserServiceImpl implements UserService{
    // 业务层调用dao层
    // 组合
    private UserDao userDao;

    // 利用set进行动态实现值的注入
    public void set(UserDao userDao) {
        this.userDao = userDao;
    }

    @Override
    public void getUser() {
        userDao.getUser();
    }
}



import com.wang.dao.UserDao;
import com.wang.dao.UserDaoMysqlImpl;
import com.wang.service.UserService;
import com.wang.service.UserServiceImpl;

public class MyTest {

    public static void main(String[] args) {
        // 用户实际调用的是业务层，dao层不需要其接触
        UserService userService = new UserServiceImpl();
        // 让用户选择实现
        ((UserServiceImpl) userService).set(new UserDaoMysqlImpl());
        userService.getUser();
    }
}

变化：之前的代码，用需求会影响原来的代码，会根据用户需求修改源码，会破坏程序的完整性，如果程序代码量很大，修改一次的成本代价十分昂贵。

使用set注入实现的优点：（发生了革命性的变化）

之前，程序主动创建对象，控制器在程序员手上。

使用set注入后，程序不再具有主动性，而是变成被动接收对象。

程序有主动创建对象，通过set方法被动接收对象，这叫做控制反转.

这种思想，从本质上解决问题，程序员不用在去管理对象的创建。系统的耦合性大大的降低，可以更加专注在业务上的实现。这是IOC的原型！并不是最正宗IOC。
