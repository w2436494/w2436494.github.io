用户管理操作

1、添加用户

添加一个用户

sudo useradd -m test1

其中 sudo,表示以管理员身份进行。其中-m参数表示在/home下添加用户

2 修改用户密码

sudo passwd test1

3 删除用户

sudo userdel test1

删除用户后需要删除用户目录 sudo rm -rf /home/test1

4 在登录系统时，默认不允许以root用户登录，避免使用root登录导致系统出现问题。

5 只有特殊用户，才能执行sudo,如当前可以使用w用户执行sudo,但test1不行

Linux下，把能执行sudo命令的用户称作sudoer。

2、超级用户/根用户root

类似于Windows下的Administrator用户。切换到root用户，有全部权限，可以直接使用useradd等命令。

1 首次使用需要给root用户设置一个密码

sudo passwd root 我这里设为12345678方便记忆。

2 切换到root用户

su root 其中su 表示switch user

3 退出

exit

root用户的权限很大，为了保证安全，用完后应该尽快退出root用户。

3、用户组

Linux下可以创建多个用户，可以用组进行管理用户。每个组可以设置不同的权限。

比如：

男生组 boys 。将后面四个用户添加到该组ming bo gang qiang。

1 创建组

groupadd boys

2 创建用户同时添加到组

useradd -m -g boys ming 其中-g表示添加用户 同时将用户添加到boys组

3 修改现有用户到新的组

usermod -g boys w 其中，usermod表示修改用户信息

4 查看用户和组

cat /etc/group

每一行表示一个group的信息，名称+id

5 查看用户列表

cat /etc/passwd 每一行表示一个用户

用户不多时，管理时候并不使用组的概念。useradd a1。

系统默认会给a1建立一个同名的组a1,也就是a1组只有a1用户。

4 root用户登录桌面

Ubuntu默认不允许root登录桌面环境。root设置了密码。

w@ubuntu21:~$ su root #切换到root用户
密码： 
root@ubuntu21:/home/w# gedit /etc/pam.d/gdm-autologin  #执行命令 gedit是文本编辑器程序

会进入一个文本

#%PAM-1.0
auth    requisite       pam_nologin.so
#auth	required	pam_succeed_if.so user != root quiet_success  #注释该行，保存
auth	optional	pam_gdm.so
auth	optional	pam_gnome_keyring.so
auth    required        pam_permit.so
@include common-account
# SELinux needs to be the first session rule. This ensures that any 
# lingering context has been cleared. Without this it is possible 
# that a module could execute code in the wrong domain.
session [success=ok ignore=ignore module_unknown=ignore default=bad]        pam_selinux.so close
session required        pam_loginuid.so
# SELinux needs to intervene at login time to ensure that the process
# starts in the proper default security context. Only sessions which are
# intended to run in the user's context should be run after this.
session [success=ok ignore=ignore module_unknown=ignore default=bad]        pam_selinux.so open
session optional        pam_keyinit.so force revoke
session required        pam_limits.so
session required        pam_env.so readenv=1
session required        pam_env.so readenv=1 user_readenv=1 envfile=/etc/default/locale
@include common-session
session optional        pam_gnome_keyring.so auto_start
@include common-password

在root用户下再次执行命令

gedit /etc/pam.d/gdm-password

打开后注释第三行

#auth	required	pam_succeed_if.so user != root quiet_success

关闭虚拟机重启后，即可使用root用户进行登录。

登录时选择未列出输入root,在输入密码即可。第一次进入需要初始化。

文件权限

1 文件权限查看

-ower : 文件的属主，拥有者
-r:文件是否可读read
-w:文件是否可写write
-x:文件是否可执行excute

使用ls命令查看用户权限

ls -l simple.txt

w@ubuntu21:~$ ls -l linux使用教程.md
-rw------- 1 w boys 2067  8月 11 22:11 linux使用教程.md
# -rw 表示用户权限
w@ubuntu21:~$ ls -l
#总用量 52
drwxr-xr-x 2 w boys 4096  8月 11 11:43 公共的
-rw-rw-r-- 1 w boys  135  8月 12 10:34 example.tar.gz
# 用户权限  文件里面的子项 用户名 用户所在群组  文件大小(字节)  文件创建时间  文件名
第一个字母是：
         d:该项是一个目录directory
         l:该项是一个链接link
         -:该项是一个文件
rw- r-- r--
权限三部分：自己|同组|别人分别拥有的权限（从左往右每3个字符）
如：
d rwx rwx r-x #r可读 w可写  x可执行
- rw- --- ---

2文件权限修改

chmod,即change file mode 修改文件的访问权限

比如，文件simple.txt,想让别人也能修改

chmod o+w simple.txt其中o表示other,+w表示增写权限

chmod o-w simple.txt 减去写权限

chmod a+w simple.txt  #所有人添加w权限
chmod a-w simple.txt  #所有人减去w权限

chmod u+w simple.txt #仅仅用户user自己添加w权限

注：只有文件的属主或者root用户可以修改文件权限

省略写法：

chmod +w simple.txt

chmod -w simple.txt 默认是修改自己和本组的权限。

3修改文件属主

chown,即change owner修改文件的属主

一般来说，每个用户只操作自己的用户目录，所以chown命令不常用。

如：在/opt下创建一个目录software ,将其分配给w用户

w@ubuntu21:~$ su root
密码： 
root@ubuntu21:/home/w# mkdir /opt/software
root@ubuntu21:/home/w# ls -ld /opt/software
drwxr-xr-x 2 root root 4096  8月 12 19:54 /opt/software
root@ubuntu21:/home/w# chown -R w /opt/software
root@ubuntu21:/home/w# ls -ld /opt/software
drwxr-xr-x 2 w root 4096  8月 12 19:54 /opt/software

只有文件的owner 和root 才有权限修改属主。

脚本程序

脚本Script，一种解释执行的程序。

Linux下常见的三种脚本程序。

Shell脚本 *.sh

Per脚本 *.pl

Python脚本 *.py

脚本程序，本质是一个文本文件。且具有可执行的权限。

脚本程序的都是由解释器执行的

shell脚本解释器：/bin/sh

Per脚本解释器: /bin/perl

Python脚本解释器: /bin/python3

执行一个脚本时，下面两种方式等效

./hello.py
/bin/python3 hello.py

1 shell脚本

shell脚本，就是按shell语法写出来的脚本，是Linux自带的脚本语言。

相当于Windows下的dos批处理脚本。

1 编辑文本文件，保存为.sh

#！/bin/sh
echo "hello world"

2 添加可执行权限

chmod +x hello.sh

3 执行程序

./hello.sh

注：

第一行必须声明解释器

#!/bin/sh

必须有x权限，才能够执行

执行程序时，必须加上路径

./hello.sh
/home/w/hello.sh

2 python脚本

python脚本，比shell脚本强大。

1 编辑一个文本文件 ，保存为.py

第一行加上解释器路径 #!/bin/python3

2 添加可执行权限

chmod +x hello.py

3 执行程序

./hello.py

2环境变量

以下命令在终端执行
定义环境变量
export OUTDIR=/opt/
显示环境变量
echo ${OUTDIR}
查看所有环境变量
printenv

环境变量的使用：

可以在当前终端中使用

可以在shell脚本中使用

在命令行定义的环境变量，只对当前shell终端有效，当关闭终端后，重启就消失了。如果需要让环境变量在重启后也能使用，可以定义为用户环境变量。

3用户环境变量

用户环境变量：定义在~/.profile中

在主目录下 ls -la 其中，a表示all,显示所有文件

注：在Linux下，以.开头的文件为隐藏文件

ls -la/ls -a
-rw-r--r--  1 w    boys  807  8月 11 10:51 .profile

gedit .profile可以打开文本编辑器。

进入后可以在文本最后添加用户环境变量：如：

# 定义一个java_home的环境变量
export JAVA_HOME =/opt/jdk1.8

设置完后注销用户，再重新登录才能生效。

原理：当用户打开终端时，会自动运行.profile，将变量注入当前环境中。

用户环境变量只对当前用户有效，因为每个用户都有自己的.profile文件

有的linux系统上，使用.bash_profile。

4系统环境变量

系统环境变量对所有用户对有效。

系统环境变量：定义在/etc/profile中

以root身份运行gedit /etc/profile

但，一般不直接修改/etc/profile,而是在/etc/profile.d/创建一个自定义脚本。

root@ubuntu21:/home/w# gedit /etc/profile.d/myprofile.sh

注销重新登录，就会生效。

5 path环境变量

path环境变量是使用最多的一个。用于描述可执行程序的搜索路径。

echo $PATH

多个路径之间以冒号隔开

/usr/local/bin/:/usr/local/sbin/:/usr/bin/:/usr/sbin/

系统默认从以下目录搜索可执行程序

/usr/bin  比如，/user/bin/tar(tar命令存放于/usr/bin/下)
/usr/sbin 比如,/user/sbin/useradd
/usr/local/bin
/usr/local/sbin
其中,sbin为超级用户root才能执行的程序
/usr/下系统自带的程序，/usr/local/是用户安装的程序。

修改PATH环境变量：

1 root用户编辑/etc/profile.d/myprofile.sh

设定 path环境变量

export PATH=$PATH:/opt/tomcat/bin $取出原来的值，再在其后添加一个

2 注销当前用户，重新登录生效。
